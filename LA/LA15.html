15.1 Modern Mechanism — Released Contract만 의존한다

LA14는 Boundary가 곧 Contract라는 것을 보여줬다. LA15는 그 Contract를 개발자 경험 차원에서 강제한다.

Business Capability
        ↓
Released Contract
        ↓
API State          (Contract를 운영하기 위한 정책 -- 기능이 아니다)
        ↓
Implementation

ABAP Cloud 개발 모델에서 개발자는 시스템의 모든 내부 Object에 접근할 수 없다 — 오직 API State가 "Released"로 지정된 대상(CDS View, Class, BAdI 등)만 참조할 수 있다.

" Strict Mode — Released가 아닌 Object 참조 시 문법 검사 단계에서 차단
API State: Released      → 참조 가능, 장기 호환성 보장
API State: Internal      → 참조 불가 (ABAP Cloud 개발 모델 안에서)
API State: Deprecated     → 참조 가능하나 향후 제거 예정 경고

이 강제는 언어 기능 제한이 아니라, 어떤 대상을 "의존해도 되는 계약"으로 승격시킬 것인가를 SAP와 개발자 사이에서 명시적으로 합의하는 절차다.

15.2 Why It Solves Today's Constraints — Freedom에서 Stability로

이 변화의 본질은 개발자가 시스템을 신뢰하는 방식의 전환이다. 그 전환은 한 단계가 아니라 세 단계로 이어진다 — Freedom → Predictability → Stability. 현대 Enterprise가 원하는 것은 "더 빠른 것"이 아니라 "예측 가능한 것"이고, Cloud가 선호되는 이유도 결국 이 Predictability 때문이다.

Legacy 신뢰 방식 "필요하면 내부 구현까지
들어가면 된다"
→
Modern 신뢰 방식 "공개된 Contract가
유지될 것을 신뢰한다"

LA11에서 본 것처럼, 과거에는 표준 Object를 직접 열어 확인하고 필요하면 수정할 수 있었다. 이제는 그 Object 내부가 어떻게 바뀌든, Released Contract만 지켜지면 Extension은 영향을 받지 않는다. 이것이 LA14가 약속한 Independent Lifecycle을 기술적으로 실현하는 방법이다 — Contract가 Freedom을 Stability로 교환해 준다.

15.3 New Trade-offs — 신뢰의 대가

영역Trade-off
Freedom → Stability 교환 내부 구현을 직접 들여다보고 우회할 수 있던 자유를 내려놓는 대신, 장기적인 호환성 약속을 얻는다
Curated API의 한계 Released API가 아직 모든 내부 기능을 다 대체하지 못해, 일부 요구사항은 당장 대응이 어려울 수 있음(LA14의 제한된 확장 범위와 동일한 구조)
사고방식 재훈련 "동작하는 코드"가 아니라 "허용된 계약 안에서 동작하는 코드"를 짜는 습관으로 개발자가 재훈련되어야 함
Innovation Speed vs Platform Consistency 오늘 마음대로 만들 수 있는 자유보다, 5년 후에도 Upgrade 가능한 상태가 더 중요해짐 — 가장 근본적인 Trade-off
Architecture Misconception Warning ABAP Cloud는 "클라우드 전용으로 축소된 ABAP"가 아니다. Cloud는 이 모델이 가장 먼저 강제되는 환경일 뿐, 본질은 안전하게 진화해야 하는 모든 시스템에 적용되는 의존성 규율이다.

ABAP Cloud restricts what you depend on, not what you can build.
The goal is not fewer extensions, but safer evolution.
Architecture Reading Point LA14가 Boundary를, LA15가 그 Boundary의 실체인 Contract를 설명했다. 다음 챕터인 LA16(RAP)에서는 이 Contract가 구체적으로 어떤 형태를 가져야 하는지 — Business Object — 가 등장한다.
