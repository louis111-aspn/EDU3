5.1 Legacy Origin вҖ” RDBMS Normalization

н•ҳлӮҳмқҳ нҡҢкі„ м „н‘ң(Accounting Document)лҠ” Header м •ліҙ(нҡҢмӮ¬мҪ”л“ң, м „н‘ңлІҲнҳё, нҡҢкі„м—°лҸ„, м „кё°мқј, м „н‘ңмң нҳ•)мҷҖ лӢӨмҲҳмқҳ Line Item(кі„м •, кёҲм•Ў, Cost Center, м°ЁлҢҖкө¬л¶„)мңјлЎң кө¬м„ұлҗңлӢӨ. Header м •ліҙлҘј лӘЁл“  Line Item н–үл§ҲлӢӨ л°ҳліө м ҖмһҘн•ҳл©ҙ к°ҷмқҖ к°’мқҙ мҲҳмӢӯ~мҲҳл°ұ лІҲ мӨ‘ліөлҗңлӢӨ.

1980~90л…„лҢҖ RDBMS м„Өкі„ мӣҗм№ҷмқҖ мқҙлҹ° л°ҳліөмқ„ н—Ҳмҡ©н•ҳм§Җ м•ҠлҠ”лӢӨ. SAPлҠ” HeaderмҷҖ Itemмқ„ лі„лҸ„ TableлЎң л¶„лҰ¬н•ҙм„ң м ҖмһҘн–ҲлӢӨ вҖ” мқҙлҠ” лӢӨмқҢ м„ё к°Җм§ҖлҘј лҸҷмӢңм—җ лӢ¬м„ұн•ҳкё° мң„н•ң м„ нғқмқҙм—ҲлӢӨ.

  • RDBMS Normalization вҖ” л°ҳліө лҚ°мқҙн„°лҘј м ңкұ°н•ҙ м •к·ңнҷ” мӣҗм№ҷ мӨҖмҲҳ
  • Data Redundancy Reduction вҖ” Header м •ліҙ мӨ‘ліө м ҖмһҘмқ„ н”јн•ҙ лҸҷмқј м •ліҙмқҳ л°ҳліө м ҖмһҘмқ„ мөңмҶҢнҷ”
  • Data Integrity вҖ” Header к°’ ліҖкІҪ мӢң н•ң кііл§Ң мҲҳм •н•ҳл©ҙ лҗҳлҠ” лӢЁмқј м§„мӢӨ м§Җм җ нҷ•ліҙ

BKPFлҠ” м „н‘ңлӢ№ н•ҳлӮҳмқҳ н–үл§Ң к°Җм§Җкі , BSEGлҠ” м „н‘ңлӢ№ м—¬лҹ¬ н–ү(Line Item мҲҳл§ҢнҒј)мқ„ к°Җм§„лӢӨ. н•ҳлӮҳмқҳ м „н‘ңлҘј мҷ„м „нһҲ мқҪмңјл Өл©ҙ н•ӯмғҒ BKPFмҷҖ BSEGлҘј KeyлЎң Joinн•ҙм•ј н•ңлӢӨ. ECC мӢңлҢҖ SAP Application лҢҖл¶Җл¶„мқҖ мқҙмҷҖ к°ҷмқҖ Header-Item кё°л°ҳ Transaction Persistence Patternмқ„ мӮ¬мҡ©н–ҲлӢӨ вҖ” CO мҳҒм—ӯмқҳ COEP, Open/Cleared Item Indexмқё BSIS/BSAS л“ұмқҙ лҢҖн‘ңм ҒмқҙлӢӨ.

5.3 Transaction Consistency вҖ” Lock / Commit / Referential Integrity

Header-Item л¶„лҰ¬лҠ” м ҖмһҘ нҡЁмңЁл§Ңмқ„ мң„н•ң кІғмқҙ м•„лӢҲм—ҲлӢӨ. м „н‘ң м ҖмһҘ(Posting) мӢң м—¬лҹ¬ Line Itemмқҙ н•ҳлӮҳмқҳ Header м•„лһҳ мӣҗмһҗм ҒмңјлЎң м»Өл°Ӣлҗҳм–ҙм•ј н•ңлӢӨлҠ” Transaction мҡ”кө¬мӮ¬н•ӯкіјлҸ„ л§һл¬јл Ө мһҲлӢӨ.

  • Lock вҖ” м „н‘ң Posting мӨ‘ Header/Itemмқҙ лҸҷмӢңм—җ Lockлҗҳм–ҙ л¶Җл¶„ м ҖмһҘмқ„ л°©м§Җ
  • Commit вҖ” HeaderмҷҖ лӘЁл“  Itemмқҙ н•ҳлӮҳмқҳ LUW(Logical Unit of Work) м•Ҳм—җм„ң н•Ёк»ҳ Commit
  • Referential Integrity вҖ” SAP Business Transactionмқҳ н•өмӢ¬ м •н•©м„ұмқҖ Application Layer Logicкіј SAP Transaction Control Frameworkм—җ мқҳн•ҙ кҙҖлҰ¬лҗҳм—Ҳмңјл©°, м „нҶөм Ғмқё Database Foreign Key мӨ‘мӢ¬ м„Өкі„мҷҖлҠ” лӢӨлҘё м ‘к·јмқҙм—ҲлӢӨ

5.4 Modern Challenge вҖ” Real-time Analytics мӢңлҢҖмқҳ л№„мҡ©

мҳҒм—ӯл№„мҡ©
Join Complexity м „н‘ң н•ҳлӮҳлҘј мҳЁм „нһҲ мЎ°нҡҢн•ҳл Өл©ҙ BKPF+BSEG(+COEP л“ұ кҙҖл Ё Table)лҘј н•ӯмғҒ Joinн•ҙм•ј н•Ё
Real-time Analytics лҢҖлҹүмқҳ Item лҚ°мқҙн„°лҘј мӢӨмӢңк°„мңјлЎң м§‘кі„н•ҳл Өл©ҙ Join л№„мҡ©мқҙ лҲ„м Ғлҗҳм–ҙ мқ‘лӢө мҶҚлҸ„м—җ мҳҒн–Ҙ
Parallel Processing Header-Item кө¬мЎ°м—җм„ңмқҳ лі‘л ¬ м§‘кі„лҠ” Row кё°л°ҳ RDBMS нҷҳкІҪм—җм„ң м¶”к°Җм Ғмқё мІҳлҰ¬ л№„мҡ©мқ„ лҸҷл°ҳ
Architecture Misconception Warning Header-ItemмқҖ мһҳлӘ»лҗң лӘЁлҚёмқҙ м•„лӢҲлӢӨ. Header-Item кө¬мЎ°лҠ” OLTP нҷҳкІҪм—җм„ң Transaction IntegrityмҷҖ Update EfficiencyлҘј нҷ•ліҙн•ҳкё° мң„н•ң н•©лҰ¬м Ғмқё м„Өкі„мҳҖлӢӨ. л¬ём ңлҠ” к°ҷмқҖ лӘЁлҚёмқ„ Real-time AnalyticsмҷҖ Enterprise Reporting мҡ”кө¬к№Ңм§Җ к·ёлҢҖлЎң нҷ•мһҘн•ҳл Ө н–Ҳмқ„ л•Ң л°ңмғқн•ң Architecture MismatchлӢӨ.

вҶ“ HANA Columnar Store мң„м—җм„ң нҶөн•©
ACDOCA вҖ” Universal Journal FI(BSEG) + CO(COEP) + AA + ML Line Itemмқ„ н•ҳлӮҳмқҳ н…Ңмқҙлё”лЎң нҶөн•©

S/4HANAлҠ” Header-Item лӘЁлҚёмқ„ "нҸҗкё°"н•ң кІғмқҙ м•„лӢҲлӢӨ. BKPFлҠ” м—¬м „нһҲ мЎҙмһ¬н•ҳл©° Header м •ліҙлҘј лӢҙлҠ”лӢӨ. ліҖнҷ”лҠ” Line Item кі„м—ҙ Tableл“Ө вҖ” кіјкұ° FI(BSEG)В·CO(COEP)В·AAВ·MLлЎң нқ©м–ҙм ё мһҲлҚҳ лҚ°мқҙн„°лҘј HANAмқҳ Columnar In-Memory кө¬мЎ° мң„м—җм„ң ACDOCA н•ҳлӮҳлЎң нҶөн•©н•ң кІғмқҙлӢӨ. Join л№„мҡ©мқҙ л¬ём ңмҳҖлҚҳ мһҗлҰ¬м—җ, HANAлҠ” Column лӢЁмң„ м§‘кі„лЎң лҢҖмқ‘н•ҳкі , лҚ°мқҙн„° лӘЁлҚёмқҖ м—¬лҹ¬ н•ҳмң„мӣҗмһҘмқ„ н•ҳлӮҳмқҳ Universal JournalлЎң л¬¶м—ҲлӢӨ.

мқҙ м—ӯмӢң "Normalizationмқ„ лІ„лҰ¬лҠ” кІғ"мқҙ м•„лӢҲлқј, OLTP м •н•©м„ұмқ„ мң м§Җн•ҳл©ҙм„ң OLAP м§‘кі„ л№„мҡ©мқ„ Hardware(HANA In-Memory Columnar)мҷҖ Data Model нҶөн•©мңјлЎң н•Ёк»ҳ нқЎмҲҳн•ҳлҠ” л°©н–ҘмқҙлӢӨ. лӢӨл§Ң мқҙ ліҖнҷ”лҠ” Financial Actual Data мҳҒм—ӯм—җм„ң м—¬лҹ¬ Application IndexмҷҖ Aggregate Tableмқ„ м ңкұ°н•ҳкі  Single Journal кё°л°ҳ м ‘к·јмқ„ к°•нҷ”н•ң кІғмқҙм§Җ, SAP м „мІҙк°Җ Header-Item кө¬мЎ°лҘј нҸҗкё°н•ң кІғмқҖ м•„лӢҲлӢӨ вҖ” Sales Order, Purchase Order, Delivery л“ұ лӢӨлҘё Business ObjectлҠ” м—¬м „нһҲ Header-Item кө¬мЎ°лҘј мң м§Җн•ңлӢӨ.

Architecture Reading Point LA05лҠ” Database Architecture м¶•мқҳ мІ« мӮ¬лЎҖлӢӨ. к°ҷмқҖ м¶•мқҳ лӢӨмқҢ мӮ¬лЎҖлҠ” LA06(Batch Processing мӨ‘мӢ¬ мҡҙмҳҒ)мқҙл©°, Header-Item кө¬мЎ°к°Җ л§Ңл“Өм–ҙлӮё Join л№„мҡ©мқҙ мҷң кіјкұ°м—җлҠ” Batch(м•јк°„ мІҳлҰ¬)лЎң нқЎмҲҳлҗ  мҲҳл°–м—җ м—Ҷм—ҲлҠ”м§ҖлҘј лӢӨлЈ¬лӢӨ.
One Sentence Takeaway
SAP Architectureмқҳ ліҖнҷ”лҠ” кіјкұ° м„Өкі„лҘј л¶Җм •н•ҳлҠ” кіјм •мқҙ м•„лӢҲлқј, Transaction мӨ‘мӢ¬ мөңм Ғнҷ”лҘј Real-time Business PlatformмңјлЎң нҷ•мһҘн•ҳлҠ” м§„нҷ” кіјм •мқҙлӢӨ.
